Slam and Navigation¶

Common nodes to run slam or navigation of the robot

Open a terminal

If working with a user workspace (previously compiled in the Raspberry)

source path_user_workspace/devel/setup.bash

if not working with a user workspace

source /home/pi/robotics_ws/master_ws/devel/setup.bash

Launch gopigo3_node

roslaunch gopigo3_node gopigo3.launch

Open another terminal and source the workspace

source path_workspace/setup.bash

Launch lidar node (depending on the lidar mounted on the robor blue lidar (ydlidar), black lidar (rplidar))

roslaunch ydlidar lidar.launch (or roslaunch rplidar_ros rplidar.launch)

Optional (Launch camera node) To see images of the camera

Open another terminal and source the workspace

source path_workspace/setup.bash

roslaunch raspicam_node camerav2_1280x960_10fps.launch enable_raw:=true camera_frame_id:="laser_frame"

Slam nodes¶

To move the robot around the maze to create the map

Open another terminal and source the workspace

source path_workspace/setup.bash

rosrun key_teleop key_teleop.py /key_vel:=/cmd_vel

or

source path_workspace/setup.bash

roslaunch wall_follower wall_follower.launch

To generate the map

Open another terminal and source the workspace

source path_workspace/setup.bash

roslaunch gopigo_slam gopigo3_slam_stl.launch

To view the map while moving the robot and generating the map

Open another terminal and source the workspace

source path_workspace/setup.bash

rviz

In rviz open a gmapping configuration file from

/home/pi/robotics_ws/master_ws/src/gopigo_slam/rviz/gmapping.rviz

When the map is complete, to save the map with name map_name

rosrun map_server map_saver -f  /home/pi/robotics_ws/master_ws/src/gopigo_slam/maps/map_name

or

cd /home/pi/robotics_ws/master_ws/src/gopigo_slam/maps/
rosrun map_server map_saver -f  /home/pi/robotics_ws/master_ws/src/gopigo_slam/maps/map_name

A map_name.pgm and a map_name.yaml will be created in the maps folder

Navigation nodes¶

To navigate in a previously saved map

The file /home/pi/robotics_ws/master_ws/src/gopigo_slam/launch/gopigo3_navigation_stl.launch or gopigo3rp_navigation_stl.launch has to be modified to point to the map that wants to be used for navigation

<arg name="map_file" default="$(find gopigo_slam)/maps/map_name.yaml"/>

After the previous modification of the navigation file to point the desire map

Open another terminal and source the workspace

source path_workspace/setup.bash

roslaunch gopigo_slam gopigo3_navigation_stl.launch or roslaunch gopigo_slam gopigo3rp_navigation_stl.launch

To autonomous navigate in the map

Open another terminal and source the workspace

source path_workspace/setup.bash

rviz

In rviz open a gmapping configuration file from

/home/pi/robotics_ws/master_ws/src/gopigo_slam/rviz/navigation.rviz

Use the Initial Pose to place the robot in the map

Use the navigation goal to send the robot to a place in the map
